GIT – Einführungsonepager

Was ist GIT?

GIT ist ein verteiltes Versionskontrollsystem, das es Entwicklern ermöglicht, Änderungen an Dateien und Projekten zu verfolgen und zu verwalten. Es wurde von Linus Torvalds entwickelt und ist besonders beliebt in der Softwareentwicklung, da es mehrere Programmierer unterstützt, die gleichzeitig an einem Projekt arbeiten können.

Wozu dient GIT?

  • Versionskontrolle: Verfolgt alle Änderungen an Dateien, sodass frühere Versionen wiederhergestellt werden können.
  • Zusammenarbeit: Erlaubt mehreren Entwicklern, an einem Projekt zu arbeiten, ohne dass Konflikte auftreten.
  • Branching: Unterstützt das Arbeiten an verschiedenen Funktionen oder Fehlerbehebungen in separaten "Branches".

Anleitung zur Nutzung von GIT

1. Nutzung via Kommandozeile

  1. GIT installieren:
    • Auf Windows: GIT für Windows herunterladen und installieren.
    • Auf macOS: brew install git (Homebrew muss installiert sein).
    • Auf Linux: sudo apt-get install git (Debian/Ubuntu).
  2. GIT konfigurieren: 
    git config --global user.email "deine.email@example.com"
 git config --global user.name "Dein Name"
  3. Ein neues Repository erstellen (Klonen):

    Repository erstellen

     
    mkdir mein-projekt
cd mein-projekt
git init

    Repository klonen

     
    mkdir mein-projekt
cd mein-projekt
git clone "Repository-url"
  4. Dateien hinzufügen und committen: 
    echo "Hello World" > datei.txt
git add datei.txt
git commit -m "Erstes Commit"
  5. Änderungen pushen: 
    git push
  6. Änderungen pullen: 
    git pull

2. Nutzung via Visual Studio Code

  1. VS Code installieren: Lade Visual Studio Code von der offiziellen Website herunter und installiere es.
  2. GIT in VS Code aktivieren: Stelle sicher, dass GIT installiert ist und VS Code es erkennen kann.
  3. Ein neues Repository erstellen:
    • Öffne VS Code und erstelle einen neuen Ordner.
    • Klicke auf das Quellcodeverwaltungs-Icon (Symbol mit zwei Verzweigungen) und wähle "Neues Repository".
  4. Dateien hinzufügen und committen:
    • Füge neue Dateien im Explorer-Bereich hinzu.
    • Klicke auf das "+"-Symbol neben der Datei im Quellcodeverwaltungsbereich, um sie hinzuzufügen.
    • Gib eine Commit-Nachricht in das Eingabefeld ein und klicke auf den Haken.
  5. Änderungen pushen: 
    git push
  6. Änderungen pullen: 
    git pull

Übungsaufgaben zu GIT

Aufgabe 1: Repository klonen

Klonen Sie eine externe Repository auf ihr Gerät.

mkdir mein-repo
cd mein-repo
git clone https://github.com/NiklasSpelters/Webtechnologie.git

Aufgabe 2: Commit und Push

Füge eine Datei index.html hinzu und committe diese. Push die Änderungen zu einem Remote-Repository.

echo "<h1>Hallo Welt</h1>" > index.html
git add index.html
git commit -m "index.html hinzugefügt"
git push

Aufgabe 3: Änderungen pullen

Simuliere, dass Änderungen im Remote-Repository vorgenommen wurden. Führe git pull aus, um diese Änderungen zu holen.

git pull
